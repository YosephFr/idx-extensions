# Publishing Guidelines

## Scope

- High-level rules for publishing extension examples/contracts.

## Source Files

- `docs/plugin-development-guide.md`
- `docs/store-theme-guide.md`
- `docs/booking-theme-guide.md`

## Rules

- Keep examples browser-safe.
- Do not include production secrets.
- Keep manifests explicit.
- Keep docs aligned with runtime contracts in platform repos.
- Version example snapshots through directory or manifest changes.

## Update Triggers

- Review pipeline changes.
- Public publishing process changes.
- New artifact type.
